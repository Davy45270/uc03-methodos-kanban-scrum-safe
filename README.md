# UC03 Méthodos Kanban/Scrum/SAFe

## Expression de besoin fonctionnelle
- **Objectifs** : supporter plusieurs méthodologies (Kanban, Scrum, SAFe) avec des workflows adaptés.
- **Acteurs** : Scrum Master, RTE, équipes agiles, managers.
- **Flux clés** : paramétrage boards → rituels (sprint planning, review, PI planning) → suivi WIP → rétros.
- **Exceptions** : équipes hybrides; changement de méthode en cours de cycle.
- **Critères d’acceptation** : boards configurables, métriques natives (velocity, WIP), templates par méthodo.

## Analyse ROI
- **Gains** : adoption rapide, réduction de l’admin, amélioration du flux de livraison.
- **Coûts** : configuration initiale, coaching agile, formation.
- **KPI cibles** : throughput, WIP moyen, respect des engagements sprint/PI, lead time.
